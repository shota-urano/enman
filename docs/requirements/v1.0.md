# 要件定義書 - 夫婦共有家計簿アプリ（enmann）

- バージョン: v1.0
- 作成日: 2025-08-31
- 想定プラットフォーム: スマホ向けWeb（PWA化なし／PC表示はMVP対象外）
- 技術構成: Next.js + Supabase（Auth/DB/Storage）
- 想定対象: 夫婦2名で1世帯を共有（1ユーザー=1世帯）

## 背景・目的
- 家計の透明性とコミュニケーションを両立し、ストレスなく継続利用できる夫婦向け家計簿を、無料枠で軽量に実現する。
- 市場アプリに不足する「相互リアクション（コメント／スタンプ）」「サブスク起点の登録」「シンプルで速い家計把握」を満たす。

## ステークホルダー
- エンドユーザー: 夫、妻（世帯内は双方フルアクセス）
- 開発者／運用者: プロダクトオーナー（個人開発、Vercel + Supabase 無料枠運用）
- 将来関係者: デザイナー／QA（現時点では不在、将来拡張想定）

## 機能要件
- 認証・世帯: Supabase Auth（メール／パスワード or 魔法リンク）。世帯（家計スペース）作成／招待機能。
- 収入／支出登録: 金額・カテゴリ・支払元／収入元・日付・（支出のみ）場所・メモ。
- リアクション: コメント（テキスト）、スタンプ（絵文字リアクション）。
- サブスク管理: 請求／入金予定の事前通知→金額確定→当月分登録。
- カレンダー／集計: 月表示、日別合計、月次サマリー（カテゴリ別可視化）。
- 通知: アプリ内通知のみ（プッシュは対象外）。
- 設定: 月次締め日（ユーザー設定）。

## 基本機能
- 初回フロー: ユーザー登録→世帯作成→招待リンク共有→相手参加。
- 取引登録: 収入／支出の新規登録（3タップ目安）、入力内容はローカル下書き自動保存。
- 明細画面: 明細閲覧、相互コメント／スタンプ、編集・削除（本人のみ）。
- サブスク: 定義一覧→予定日前日21:00に通知→金額確認→当月確定登録。
- カレンダー: 月表示、日タップで明細、月次サマリー（カテゴリ別円／棒・合計）。
- 設定: 締め日（1〜31／末尾丸め）、カテゴリ・アカウント（支払元／収入元）編集、招待管理。

## 詳細仕様
### 認証／世帯
- 認証: Supabase Auth（外部IdPなし）。セッション維持は標準を使用。
- 世帯: 1ユーザー=1世帯前提。招待リンクで相手を1名招待（最大2名）。複数世帯参加はMVP範囲外。

### 取引モデル
- 必須項目: 種別（収入／支出）、日付、金額（JPY・整数）、カテゴリ、支払元／収入元、作成者。
- 任意項目: 場所（支出のみ）、メモ。
- カテゴリ: 初期カテゴリを提供し、ユーザーが追加／編集可能。
- アカウント（支払元／収入元）: 初期に「口座」「クレジットカード」を提供。ユーザーが追加／編集可能。

### コメント／スタンプ
- コメント: テキストのみ。作成者のみ編集／削除可。相手にアプリ内通知。
- スタンプ: 絵文字リアクション。1ユーザーにつき各取引へ1種×1回まで（再押下で取消／差し替え可）。

### サブスク
- 定義: 名称、想定金額、カテゴリ、アカウント、請求／入金日、メモ。
- 通知: 予定日の前日21:00にアプリ内通知（繰返しは月次固定）。
- 登録: 通知から金額確認→当月分として確定登録。金額は編集可能（変動対応）。

### カレンダー／集計
- カレンダー: 月表示。各日に「支出合計／収入合計／差額」を表示。日タップで明細一覧。
- 月次サマリー: カテゴリ別円／棒グラフ、月合計（収入／支出／差額）。締め日設定を反映。

### 設定
- 締め日: 1〜31日から選択。該当月に存在しない日（例: 31日）は月末に丸める。
- ローカライズ: 通貨JPY、小数なし。言語=日本語固定。タイムゾーン=Asia/Tokyo。

### 監査／権限
- 権限: 世帯内は双方フルアクセス（閲覧・登録・編集・削除）。ロール（閲覧のみ等）はMVP外。
- 監査: 取引／コメントの作成者・更新者・日時を記録（簡易）。

## 非機能要件
- 可観測性／ログ: アプリ内エラー通知（トースト）。バックエンドはSupabase標準ログを利用。
- データ整合性: 同期競合は「後勝ち」。重大衝突は最終保存者優先。

## パフォーマンス
- 応答時間（P95／P99）
  - 一覧表示（カレンダー・月次サマリー含む）: ≤1.0s ／ ≤2.0s
  - 新規保存（収入／支出／サブスク確定）: ≤1.0s ／ ≤2.0s
- 同期: 保存後のUI反映 ≤500ms（楽観的更新可）。
- クライアント: 入力確定から≤100msでローカル下書き保存。

## セキュリティ
- 認証／認可: Supabase Auth。世帯スコープでRLS（Row Level Security）を適用し第三者アクセスを防止。
- データ保護: 通信はTLS。保管はSupabase標準暗号化に依存。
- 監査: 作成／更新／削除のメタデータ保持（ユーザーID、タイムスタンプ）。

## 可用性
- 目標: 無保証（無料枠）。ダウン時の通知／自動復旧体制は持たない。
- バックアップ／RPO／RTO: Supabase Free前提のためSLA・自動バックアップ要件なし。データ消失リスクは受容（同意前提）。

## 制約事項
- 技術: Next.js + Supabase（Auth/DB/Storage）。PWA／プッシュ通知はMVP外。
- 運用: Vercel + Supabase Free枠内で運用。上限到達時は手動アーカイブ／削除で対応。
- 時間／予算: 1ヶ月以内、追加コストなし。
- 対象外（MVP外）: CSVエクスポート／インポート、複数世帯参加、週／年ビュー、外部IdP、PCレイアウト、添付ファイル。

## 受け入れ条件
- A1: 新規登録→世帯作成→招待→相手が参加できること。
- A2: 双方が収入／支出を登録でき、日次／月次集計に即時反映されること。
- A3: 取引に相手がコメント／スタンプでき、相手にアプリ内通知が届くこと。コメントは本人のみ編集／削除できること。
- A4: サブスク予定定義が可能で、予定日前日21:00にアプリ内通知が届き、金額確認から当月分の確定登録ができること。
- A5: カレンダー月表示で日毎の収入／支出／差額が表示され、日タップで明細が見られること。
- A6: 月次サマリーでカテゴリ別グラフと合計が表示され、締め日設定の変更が反映されること。
- A7: 主要操作の応答時間（一覧／保存）がP95 1.0s、P99 2.0s以内であること。
- A8: オフライン時に入力した内容がローカル下書きとして保持され、復帰時に後勝ちで同期されること。
- A9: 権限は世帯内フルアクセスで、外部ユーザーからのアクセスがRLSで遮断されること。

## 優先度・リリース計画
- MVP（Must）
  - 認証／世帯（作成・招待）、収入／支出登録、カテゴリ／アカウント初期マスタ+編集、コメント／スタンプ、サブスク通知→確定登録（アプリ内通知・前日21:00）、カレンダー月表示、月次サマリー、締め日設定、簡易監査、下書きローカル保存、RLS。
- 次点（Should／Could・次リリース候補）
  - PWA化・プッシュ通知、PCレイアウト、週／年ビュー、複数世帯参加、CSVエクスポート、外部IdP、詳細監査・ロール強化、添付ファイル。
