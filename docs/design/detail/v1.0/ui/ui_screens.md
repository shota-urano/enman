画面詳細設計（20250831_v1.0）

共通レイアウト
- ヘッダー: height: 56–60px, 固定（sticky top-0）, 背景: #FAFAFA, border-b: #E2E8F0
- メイン: padding: 16–24px（モバイル16px推奨）, max-w: 720px 中央寄せ（余白は将来）
- フッタ: futuristic-nav（LumaBar）を bottom-24〜bottom-6 に配置（キーボード開閉時は避ける）
- コンテナ: <div className="page-container pb-28">…</div>（ナビ分の下余白を確保）

1) 認証/初回セットアップ
- ルート: /auth（サインイン/アップ）, /setup（世帯作成/招待）
- レイアウト構成:
  ヘッダー: ロゴ/タイトル（center）
  本文: カード中央寄せ（mt-12）幅 360–420px
  フッタ: ナビ非表示
- コンポーネント配置:
  サインインフォーム: Email, Password, [ログイン]
  セットアップ: 世帯名, 締め日(Select), [作成] / 招待コード入力 [参加]
- 状態管理:
  ローディング: フォーム内ボタン is-loading, 入力はdisabled
  エラー: フォーム下にアラート（role=alert）
  空状態: なし
- データ表示仕様: なし
  サンプルHTML:
  <main className="page-container grid place-items-center">
    <div className="card w-full max-w-md">
      <div className="card-header text-center">サインイン</div>
      <div className="card-body space-y-4">…</div>
    </div>
  </main>

2) ホーム（月カレンダー）
- ルート: /
- レイアウト構成:
  ヘッダー: 月表示と合計サマリー（収入/支出/差額）
  サイドバー: なし（モバイル）
  メイン: 月カレンダー（6行×7列）, 行間 6px, セル min-h: 56px
- コンポーネント配置:
  カレンダー上部: 月切替（< / >）, 締め日ラベル
  日セル: 収入/支出/差額のサマリ（小さなバッジ）, 当日ハイライト
  FAB（右下）: ＋ 取引登録
- 状態管理:
  ローディング: 月全体スケルトン（セルスケルトン）
  エラー: カレンダー領域内に再試行ボタン
  空状態: 「今月の記録はまだありません」＋[取引を追加]
- データ表示仕様:
  セル内: 上段=支出(-), 下段=収入(+), 角右=差額
  サマーカード: 当月合計（支出/収入/差額）
  クリック: 日をタップで日別明細（モーダル）

3) 取引登録
- ルート: /tx/new
- レイアウト: ヘッダー「取引を追加」, メインフォーム, フッタナビ有
- コンポーネント配置:
  1行目: 種別セグメント（支出/収入）
  2行目: 金額（inputMode=numeric）+ 日付（DatePicker）
  3行目: カテゴリ（Select）
  4行目: アカウント（Select）
  5行目: （支出時のみ）場所（text）
  6行目: メモ（textarea）
  最下: [保存]（btn-primary, w-full）
- 状態管理:
  ローディング: 保存中はボタンis-loading
  エラー: トースト＋該当フィールドのエラー表示
  下書き: 入力変更から500ms以内にローカル保存（localStorage key: draft:tx）
- データ仕様:
  金額: 数値, 3桁区切り表示, マイナス不可（支出/収入で符号管理）
  日付: 当日既定, カレンダーと連動

4) 明細詳細
- ルート: /tx/:id
- レイアウト: モーダル or 画面フル（戻る）
- コンポーネント配置:
  上部: 金額/カテゴリ/日付/アカウント
  本文: メモ, 場所（支出時）
  下部: [編集][削除](本人のみ), コメント/スタンプリスト
- 状態管理:
  ローディング: スケルトン
  エラー: 取得失敗時の再読込
  アクション: 削除は確認ダイアログ
- データ表示仕様:
  金額は大きく 24px/700, 色は支出=rose-600, 収入=emerald-600

5) サブスク
- ルート: /subs
- レイアウト: リスト + 新規/編集フォーム（モーダル）
- コンポーネント配置:
  行: 名称, 金額, 周期, 次回, スイッチ（有効/無効）
  右上: [追加]
- 状態管理:
  通知: 前日21:00にプッシュ（設計のみ）
  確定: 当月分登録の確認トースト

6) レポート（月次サマリー）
- ルート: /reports
- レイアウト: タブ（サマリー/カテゴリ/アカウント）, グラフ（円/棒）, テーブル
- コンポーネント配置:
  上部: 期間セレクタ（月, 前月, 任意）
  本文: カードにグラフ, 下に明細テーブル
- 状態管理:
  ローディング: グラフスケルトン
  エラー: アラート＋再試行
- データ仕様:
  円グラフ: カテゴリ比率, テーブルと連動ハイライト

7) 設定
- ルート: /settings
- レイアウト: リスト（List/DescriptionList）
- コンポーネント配置:
  締め日（Select）, カテゴリ管理（List + CRUDモーダル）, アカウント管理, 招待管理（リンク発行/コピー）
- 状態管理:
  保存成功: 上部トースト
  削除: 確認ダイアログ
