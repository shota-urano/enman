## 1. ビジネス要求
### 1.1 背景・目的
現在様々な家計簿シェアアプリがあるが、課金が必要になったり、自分たちが欲しいような機能がなかったりするので、自分が使いやすいような家計簿アプリを作成したい
### 1.2 ビジネス目標
まずは、自分たちが使えるようなアプリを作成する
### 1.3 成功指標
自分たちが負荷なく使える

## 2. ユーザー要求
### 2.1 対象ユーザー
旦那と奥さん
### 2.2 ユーザーストーリー
**夫として：**
- 外出先でレシートを見ながら支出を簡単に登録したい
- 妻が登録した支出にスタンプで「いいね」を伝えたい
- 月末に夫婦の収支バランスを一覧で確認したい

**妻として：**
- 買い物後すぐにスマホで支出を記録したい
- 夫の登録内容にコメントで「お疲れ様」と伝えたい
- サブスクの支払い忘れを防ぎたい

**夫婦共通として：**
- お互いの金銭感覚を理解し合いたい
- 家計の透明性を保ちたい
- 簡単操作で継続して使いたい
### 2.3 ユースケース
**UC001: 支出登録**
- アクター: 夫/妻
- 前提条件: アプリにログイン済み
- 基本フロー: 
  1. 金額入力
  2. カテゴリー選択
  3. 場所入力（任意）
  4. メモ入力（任意）
  5. 登録完了

**UC002: 相手へのリアクション**
- アクター: 登録者以外
- 前提条件: 相手が支出/収入を登録済み
- 基本フロー:
  1. 登録内容を確認
  2. スタンプまたはコメントを選択
  3. 送信

**UC003: サブスク管理**
- アクター: 夫/妻
- 前提条件: サブスク情報登録済み
- 基本フロー:
  1. 通知受信
  2. 金額確認・修正
  3. 今月分として登録

## 3. 機能要求
### 3.1 必須機能（Must）
- 収入
    - 金額を登録できる
    - 収入カテゴリーを登録できる
    - 収入元を登録できる
    - メモができる
    - コメントができる(登録者じゃない相手)
    - スタンプ機能がある(デバイスの絵文字？)
- 支出
    - 金額を登録できる
    - 支出カテゴリーを登録できる
    - 支出元を登録できる
    - 場所を登録できる
    - メモができる
    - コメントができる(登録者じゃない相手)
    - スタンプ機能がある(デバイスの絵文字？)
- サブスクリプション
    - 毎月固定の支出を登録できる
        - 毎月変動で発生する光熱費などを、通知して登録漏れがないようにしたい
    - 毎月固定の収入を登録できる
        - 毎月変動で額が変わる場合は、通知して登録もれがないようにしたい
- カレンダー
    - 各日付の収支を一目でわかるようにしたい

## 4. 品質要求
### 4.1 性能要求
- **応答時間**: 画面遷移は2秒以内
- **データ同期**: リアルタイム同期（5秒以内）
- **オフライン対応**: 一時保存後、オンライン時に同期

### 4.2 使いやすさ要求
- **操作性**: 支出登録は3タップ以内で完了
- **直感性**: アイコンで機能を判断できる
- **学習コスト**: 初回利用時の説明は5分以内
- **アクセシビリティ**: 片手操作可能

### 4.3 信頼性要求
- **データ保護**: 自動バックアップ（日次）
- **同期信頼性**: データ不整合の防止
- **可用性**: 99%以上の稼働率
- **データ永続化**: 誤削除防止機能

締め日はユーザが設定できるようにして欲しいです。

4. カテゴリ/項目の粒度
